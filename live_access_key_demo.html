部屋ごとに、鍵を発行する。

招待トークンから毎回異なるQRと入域鍵を描画します。同じトークンを開き直すと、テンプレート、配色、鍵の歯、刻印まで同じ鍵へ戻ります。

デモではブラウザが安全な 8 文字トークンを生成します。本番では Firebase の Cloud Function が発行します。

Firebase room creation contract

本番の createSnapRoom は、安全な inviteToken をサーバーで発行し、deriveAccessKey(inviteToken) の戻り値を rooms/{roomId}/meta/accessKey へ保存します。ホスト画面はこのメタデータと https://ゲームURL/i/{inviteToken} を受け取り、この画面と同じ方法で描画します。参加者はQRを読み取り、Cloud Function がトークンを検証してからルームへ追加します。
